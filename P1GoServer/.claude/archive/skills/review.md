# 代码审查

当用户调用此 skill 时，对指定的代码或文件进行全面审查。

## 审查维度

### 1. 代码质量
- 代码是否清晰易读
- 命名是否有意义
- 函数/方法是否职责单一
- 是否存在重复代码
- 注释是否必要且有价值

### 2. 逻辑正确性
- 边界条件处理
- 空值/nil 检查
- 循环终止条件
- 并发安全性
- 错误处理是否完善

### 3. 性能考量
- 算法复杂度
- 内存使用
- 不必要的分配
- 数据库查询效率
- 缓存使用

### 4. Go 语言最佳实践
- error 处理是否规范
- defer 使用是否正确
- goroutine 泄漏风险
- channel 使用是否安全
- interface 设计是否合理

### 5. 安全性
- 输入验证
- SQL 注入风险
- 敏感数据处理
- 权限检查

## 输出格式

```markdown
## 代码审查报告

### 概述
[简要描述代码的功能和整体质量]

### 问题发现

#### 严重 (Critical)
- [ ] 问题描述 (文件:行号)

#### 警告 (Warning)
- [ ] 问题描述 (文件:行号)

#### 建议 (Suggestion)
- [ ] 问题描述 (文件:行号)

### 优点
- 列出代码中做得好的地方

### 总结
[整体评价和改进建议]
```

## 使用方式

- `/review` - 审查当前 git diff 的变更
- `/review path/to/file.go` - 审查指定文件
- `/review path/to/dir/` - 审查指定目录
