# 代码解释助手

当用户调用此 skill 时，帮助理解代码和技术概念。

## 解释维度

### 1. 代码级解释

对于具体代码，解释：

- **功能**: 这段代码做什么
- **输入/输出**: 参数和返回值
- **算法**: 使用的算法或逻辑
- **关键步骤**: 逐步解释主要逻辑
- **边界条件**: 特殊情况处理

```markdown
## 代码解释

### 概述
这个函数用于 [功能描述]

### 参数说明
- `param1`: 描述
- `param2`: 描述

### 执行流程
1. 首先，[步骤1]
2. 然后，[步骤2]
3. 最后，[步骤3]

### 关键点
- 第X行: 解释为什么这样做
- 第Y行: 解释这个技巧

### 示例
输入: xxx
输出: yyy
```

### 2. 架构级解释

对于系统架构，解释：

- **组件**: 主要组件及职责
- **交互**: 组件间如何通信
- **数据流**: 数据如何流转
- **设计决策**: 为什么这样设计

### 3. 概念解释

对于技术概念，提供：

- **定义**: 是什么
- **用途**: 解决什么问题
- **原理**: 如何工作
- **示例**: 具体例子
- **对比**: 与类似概念的区别

## 解释风格

### 简洁模式
```
函数X: 接收A，处理后返回B。
核心逻辑是使用算法C遍历数据。
```

### 详细模式
```
## 函数X详解

### 背景
[为什么需要这个函数]

### 详细分析
[逐行或逐块分析]

### 设计考量
[设计决策和权衡]

### 潜在问题
[可能的边界情况或注意事项]
```

## Go 语言特定解释

### 常见模式
- Context 传递模式
- Error 处理模式
- Interface 设计模式
- Goroutine 和 Channel 模式
- Defer/Panic/Recover 模式

### 标准库使用
- net/http 相关
- database/sql 相关
- encoding/json 相关
- sync 包相关

## 使用方式

- `/explain path/to/file.go` - 解释整个文件
- `/explain path/to/file.go:100-150` - 解释指定行范围
- `/explain FunctionName` - 解释特定函数
- `/explain "什么是 context"` - 解释技术概念
