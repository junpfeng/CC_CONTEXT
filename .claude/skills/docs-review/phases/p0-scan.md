# Phase 0：扫描盘点

> 目标：全面盘点目标目录下的文档，为后续审查提供基础数据。

## 步骤

1. **列出所有文档**
   - 使用 Glob 扫描目标目录下所有 `.md` 文件，在结果中过滤掉 `_review_*.md` 临时文件、`.swp`、`.bak` 等编辑器临时文件（Glob 不支持排除模式，需代码侧过滤）
   - 自动发现实际存在的子目录结构，不局限于 SKILL.md 中的标准分类
   - 获取最后修改时间：优先 `git log`；若目录不在 git 仓库中（如工作空间根目录非 git repo），使用 `stat` 命令获取文件系统时间；均不可用则填 "unknown"

2. **提取文档摘要**
   - 使用 subagent 并行读取文档（每批最多 3 个 subagent）
   - 每个 subagent 提取：标题、核心主题、关键实体（模块名/系统名）、预估行数
   - subagent prompt 必须包含：「返回精简摘要，不超过 1000 字」

3. **扫描引用关系**
   - 使用 Grep 搜索工作空间中对 `docs/` 文件的引用，构建引用图
   - 扫描范围（按优先级，前三项必做，第四项可选）：
     - **必做** `*/CLAUDE.md`、`.claude/**/*.md`：Grep pattern `docs/` 搜索相关链接
     - **必做** `docs/**/*.md`：提取所有 `[](...)` 格式的内部交叉引用
     - **必做** MEMORY.md 及记忆文件：Grep pattern `docs/` 搜索路径
     - **可选** `**/*.go`、`**/*.cs`：代码注释中的 `docs/` 引用（大项目耗时长，仅在文档数量 ≤20 时启用，或用户明确要求时启用）
   - 输出引用图：每个文档被哪些文件引用

4. **生成盘点报告**
   - 输出表格，包含以下列：

   | 文件路径 | 主题 | 关键实体 | 行数 | 最后修改 | 被引用次数 | 引用方 |
   |---------|------|---------|------|---------|-----------|-------|

   - 按子目录分组展示
   - 标注空文件或仅含索引的文件
   - **高引用文档**（被引用 ≥3 次）特别标注，变更需谨慎

## 输出

将盘点报告写入临时文件 `docs/_review_inventory.md`（Phase 1 读取此文件作为输入，Phase 3 结束后清理），同时向用户输出精简版。

## 容错

- 文件读取失败时跳过并标注
- git 信息不可用时使用文件系统时间，仍不可用则填 "unknown"
- Glob 结果为空时提示用户检查目标路径
