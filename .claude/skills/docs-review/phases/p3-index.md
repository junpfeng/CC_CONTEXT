# Phase 3：索引更新

> 目标：根据文档变更结果，重建所有索引文件，确保导航准确。

## 前置条件

- Phase 2 合并整理已完成

## 步骤

### 1. 扫描最终文件列表

- 使用 Glob 获取 `docs/` 下所有 `.md` 文件（排除 `_review_*.md` 临时文件）
- 按子目录分组

### 2. 重建子目录索引

为每个非空子目录更新 `INDEX.md`：
- 读取每个文档的标题和首段摘要
- 生成表格格式索引：

```markdown
# {子目录名} 索引

| 文档 | 说明 | 最后更新 |
|------|------|---------|
| [文件名](文件名) | 一句话摘要 | YYYY-MM-DD 或 unknown |
```

- 如子目录下无文档，删除空的 INDEX.md

### 3. 重建 docs/README.md

- 读取所有子目录 INDEX.md
- 生成顶层导航，按子目录分节，每节列出文档链接和摘要
- 保持与 CLAUDE.md 中 docs 相关引用的一致性

### 4. 验证引用完整性（必须）

Phase 2 已在变更时同步更新了引用方，此处做最终验证：
- **精确验证**：优先用 Phase 2 变更过的旧路径列表逐一 Grep 搜索，确认无残留引用
- **兜底验证**：Grep 搜索 `docs/` 路径引用中所有 `](docs/` 和 `](../docs/` 模式的链接，验证目标文件存在
- 如发现残留断链（Phase 2 遗漏），立即修复：
  - 引用目标已合并 → 更新为合并后路径
  - 引用目标已删除 → 移除引用或替换为说明文字
  - 引用目标已移动 → 更新为新路径
- 输出断链修复清单（如有）

### 5. 清理临时文件

- 删除 `docs/_review_inventory.md`
- 删除 `docs/_review_report.md`

## 输出

向用户输出最终摘要：
- 更新了哪些索引文件
- docs/ 最终文件数量和分布
- 是否有 CLAUDE.md/MEMORY.md 引用需要更新
