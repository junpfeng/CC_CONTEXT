## 调试路径规则

- **代码修改**：根据 bug 根因确定修改范围，参照对应子工程的 `.claude/rules/`
- **日志查看**：参照各子工程的日志目录和项目调试文档（如有）
- **Rules 遵守**：修改代码时，必须遵守对应子工程 `.claude/rules/` 下的规范
- **跨工程 bug**：明确标注 bug 涉及哪些工程，修复时按工程分别处理
