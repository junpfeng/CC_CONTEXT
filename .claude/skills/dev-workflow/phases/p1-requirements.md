# Phase 1：需求解析与验证

> 领域依赖：无

## 1.1 读取需求文档

Read 用户指定的需求文档，提取：功能概述、验收标准、依赖关系、涉及工程（业务/协议/配置/Rust）、优先级。

## 1.2 工程定位

读取 workspace `CLAUDE.md` 获取子工程列表，逐一验证：
1. 子工程目录是否存在
2. 子工程 `CLAUDE.md` 是否存在（存在则按需读取获取详细路径和宪法）
3. 不存在的工程标记不可用

## 1.3 依赖检查

按涉及工程检查前置条件：
- 协议修改 → protoc 编译环境就绪
- 配置修改 → 配置生成工具就绪
- DB 变更 → 数据库连接和权限
- 测试 → 测试环境或本地 mock

## 1.4 输出摘要

```
需求摘要：
- 功能名称：[name]
- 验收标准：[criteria]
- 涉及工程：[列表]
- 工程确认：各子工程可用性（目录 + CLAUDE.md）
- 依赖项：[dependencies]
- 风险点：[risks]
```

**等待用户确认后进入 Phase 2。**
