# Phase 3：任务拆解

> 领域依赖：无

## 拆解原则

- **单一职责**：每个任务只做一件事
- **工程隔离**：不同工程的任务完全隔离（可并行）
- **依赖明确**：任务间依赖关系清晰
- **可验证**：每个任务有明确完成标准

## 任务分类模板

按工程分类，从以下类别中选取适用项：

**业务工程**：基础设施（struct/接口）→ Service 层 → Manager 层 → Component 层 → 事件处理 → 单元测试 → 集成测试

**协议工程**：新增消息 → 修改枚举常量 → 代码生成 → 更新依赖

**配置工程**：新增配置表 → 修改生成器 → 运行生成 → 验证产物

**数据库**：Migration 脚本 → 本地测试 → 回滚脚本

**遗留工程参考**：阅读对标代码（仅参考，不修改）

## Wave 分组

对任务进行拓扑排序，分配 wave 编号：

- **wave 0**：无依赖的任务（协议定义、配置结构等）
- **wave N**：仅依赖 wave 0..N-1 中已完成任务的任务
- 同 wave 任务**不得操作相同文件**——拆分后列出每个 task 预期修改的文件，检查同 wave 内无交集。有交集则调整依赖将其拆到不同 wave
- 典型分组：wave 0 (Proto/配置) → wave 1 (Server + Client 并行) → wave 2 (集成/测试)

## 输出

保存到设计文档目录，包含：

1. **Wave 汇总表**：每个 wave 包含的任务、预期修改文件、并行策略
2. **任务依赖图**：Mermaid `graph TD` 格式，节点格式 `TASK-XXX["描述 (wave N)"]`，依赖用箭头连接
3. **结构化任务清单**：每行格式 `[TASK-XXX] wave:N depends:[...] project:XXX files:[...] 描述`

**输出任务清单后自动进入 Phase 4。**
