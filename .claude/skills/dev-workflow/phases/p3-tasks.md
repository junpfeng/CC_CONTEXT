# Phase 3：任务拆解

> 领域依赖：无

## 拆解原则

- **单一职责**：每个任务只做一件事
- **工程隔离**：不同工程的任务完全隔离（可并行）
- **依赖明确**：任务间依赖关系清晰
- **可验证**：每个任务有明确完成标准

## 任务分类模板

按工程分类，从以下类别中选取适用项：

**业务工程**：基础设施（struct/接口）→ Service 层 → Manager 层 → Component 层 → 事件处理 → 单元测试 → 集成测试

**协议工程**：新增消息 → 修改枚举常量 → 代码生成 → 更新依赖

**配置工程**：新增配置表 → 修改生成器 → 运行生成 → 验证产物

**数据库**：Migration 脚本 → 本地测试 → 回滚脚本

**遗留工程参考**：阅读对标代码（仅参考，不修改）

## 输出

1. **任务依赖图**：Mermaid 格式，展示工程间依赖关系
2. **结构化任务清单**：`[TASK-XXX]` 编号，按工程分组，标注依赖

保存到设计文档目录。
