# Phase 7：经验沉淀

> 领域依赖：无

## 沉淀分类

| 经验类型 | 沉淀目标 |
|----------|----------|
| 编码规范 | 对应子工程编码规范 |
| 领域知识 | 知识库中相关文档 |
| 架构约定 | 对应子工程项目说明 |
| Agent 优化 | Agent Memory |

## 流程

1. 识别经验类型 → 匹配目标文档 → Edit 追加 → 验证一致性
2. 询问用户：是否有需记录的经验？需新增/修改 Rules？需更新知识库文档？

## 自动沉淀提醒

| 触发条件 | 建议沉淀到 |
|----------|-----------|
| 新存储限制/约束 | 知识库中数据库相关文档 |
| 新接口规范 | 知识库中协议相关文档 |
| 新测试技巧/工具 | 知识库中测试相关文档 |
| 新审查检查项 | 知识库中审查相关文档 |
| 配置部署变更 | 知识库中配置相关文档 |
| 重复编码问题 | 对应子工程编码规范 |
| 新架构约定 | 对应子工程项目说明 |

## 历史案例

### 场景隔离模式

复用现有系统为新场景时的标准模式（6 步）：

1. 添加场景类型枚举
2. 添加行为控制字段（enableXxx bool + sceneType）
3. 提供场景专用构造函数
4. 在初始化分支中设置场景参数
5. 行为代码中用条件字段分支隔离
6. 协议同步用 sceneType 填充对应字段

**审查清单**：构造函数设置正确、初始化在正确分支、行为分支正确、协议填充正确、存储方法独立、原场景回归测试。
