# Phase 4：实现

> 领域依赖：无

## 实现策略

优先级：接口/协议 → 配置 → DB → 业务（确保依赖就绪后再实现业务逻辑）

> 根据项目实际工程结构调整优先级，核心原则是"被依赖的先实现"。

## 跨工程并行

启动 subagent 并行执行不同工程任务：

| 分组 | 内容 | 依赖 |
|------|------|------|
| 组1（并行） | 基础设施类任务：接口定义 / 配置结构 / DB Migration | 无 |
| 组2（并行） | 基础设施衍生：代码生成 / 配置产物生成 / DB 本地验证 | 组1 |
| 组3（串行） | 业务实现：数据结构 → 业务逻辑 → 单元测试 → 集成测试 | 组1+2 |

**约束**：
- 每个 subagent 须先 Read 设计方案理解上下文
- 涉及相同文件的任务不并行
- 遇到阻塞暂停汇报，不自行绕过

## 编码要点

实现时遵守各子工程的编码规范。额外关注：

- **事务性**：先验证后执行、失败回滚、请求 ID 追踪、关键操作日志
- **错误处理**：不忽略错误、包装上下文信息、有意义的错误消息
- **并发安全**：共享变量加锁、及时释放、避免长持锁
- **资源管理**：确保资源释放（文件句柄 / 连接 / 锁等）、超时控制
