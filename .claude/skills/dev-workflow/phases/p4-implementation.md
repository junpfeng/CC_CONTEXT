# Phase 4：实现

> 领域依赖：无

## 实现策略

优先级：协议 → 配置 → DB → 业务（确保依赖就绪后再实现业务逻辑）

## 跨工程并行

启动 subagent 并行执行不同工程任务：

| 分组 | 内容 | 依赖 |
|------|------|------|
| 组1（并行） | 协议：消息定义 / 配置：配置表 / DB：Migration 脚本 | 无 |
| 组2（并行） | 协议：代码生成 / 配置：运行生成器 / DB：本地测试 | 组1 |
| 组3（串行） | 业务：struct → Service → 单元测试 → 集成测试 | 组1+2 |

**约束**：
- 每个 subagent 须先 Read 设计方案理解上下文
- 涉及相同文件的任务不并行
- 遇到阻塞暂停汇报，不自行绕过

## 编码要点

实现时遵守各子工程的编码规范。额外关注：

- **事务性**：先验证后执行、快照回滚（defer）、请求 ID 追踪、关键操作日志
- **错误处理**：不忽略错误、wrap 上下文、有意义的错误信息
- **并发安全**：共享变量加锁、defer 释放、避免长持锁
- **资源管理**：defer 关闭资源、context 控制超时
