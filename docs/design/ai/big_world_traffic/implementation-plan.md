## 阶段划分与依赖

```
阶段1（路网适配）  ←── 所有后续阶段的基础
  │
  ├──→ 阶段2（信号灯+路口决策）
  │       │
  │       └──→ 阶段3（驾驶人格驱动信号灯/跟车行为）
  │
  ├──→ 阶段4（碰撞闪避升级：需转向控制）

  │       ↑

```

## 阶段 1：路网适配与基础寻路（P0）

> 将小镇原型（TownRoadGraph/TownPathfinder）适配到大世界 50K 路点规模。

### 任务清单

| # | 任务 | 工程 | 说明 |
|---|------|------|------|
| 1.1 | TrafficRoadGraph | 客户端 | 复用 WaypointManager 数据构建交通专用图视图 |
| 1.2 | TrafficPathfinder | 客户端 | A* 寻路（分帧+缓存+节点上限）|
| 1.3 | 路口数据提取 | 工具 | 离线工具分析路网生成 `big_world_junctions.json` |
| 1.4 | ServerRoadNetwork | 服务端 | 轻量路网索引（网格空间查询），供密度管理使用 |
| 1.5 | 巡航目标选择 | 客户端 | 随机选远处节点 + A* + 防折返 |
| 1.6 | 集成验证 | 客户端 | 替换现有路径规划，大世界场景跑通 |

### 验收标准
- 车辆在大世界场景中自主巡航，路径合理
- A* 寻路单次 < 5ms（分帧场景 < 1ms/帧）
- 无路径死循环或折返抖动

## 风险缓解

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| 50K 路点 A* 性能 | 寻路卡顿 | 分帧计算 + LRU 缓存 + 搜索上限 |
| 295 路口信号灯同步 | 网络带宽 | 仅 AOI 内广播，状态切换时才发 |
| 侧闪跑出路面 | 视觉穿帮 | 车道边界检查 + 最大偏移限制 |
| DotsCity 集成冲突 | 物理异常 | 交通 AI 控制优先，DotsCity 仅物理执行 |
| 人格参数不平衡 | 行为不自然 | 配置表驱动，可热更调参 |
